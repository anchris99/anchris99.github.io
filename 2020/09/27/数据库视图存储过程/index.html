视图

什么是视图

视图是虚拟的表。与包含数据的表不一样，视图只包含使用时动态检索数据的查询。视图就是查询，它包含了一系列的SQL语句。

可以将类比 Java中面向对象特性中的 封装， 就是屏蔽了一些细节，只暴露出一个简单了对外调用的接口。

规则

操作

  • CREATE VIEW语句来创建。

  • 使用SHOW CREATE VIEW viewname；来查看创建视图的语句。

  • DROP VIEW viewname;删除视图。

  • 更新视图：可以先用DROP再用CREATE，也可以直接用CREATE OR REPLACE VIEW。

限制

  • 视图必须唯一命名 。

  • 对于可以创建的视图数目没有限制。

  • 视图可以嵌套，即可以利用从其他视图中检索数据的查询来构造一个视图。

  • ORDER BY覆盖： ORDER BY可以用在视图中，但如果从该视图检索数据SELECT中也含有ORDER BY，那么该视图中的ORDER BY将被覆盖。

  • 视图可以和表一起使用。

视图的更新

并非所有视图都是可更新的。 如果MySQL不能正确地确定被更新的基数据，则不允许更新（包括插入和删除）。这实

如果视图定义中有以下操作，则不能进行视图的更新：

  • 分组（使用GROUP BY和HAVING）；

  • 联结 (JOIN)

  • 子查询

  • 并 (UNION UNION ALL)

  • 聚集函数（Min()、Count()、Sum()等）

视图更新的时候，不是作用在 视图上，而是通过视图 作用在了基表上， 因为 永远要记住视图只是一个虚表，是一组逻辑，真正的实体落在基表上面。

视图的作用

  1. 重用 SQL语句， 创建视图后相同的操作就可以通过视图重复使用。
  2. 简化操作， 将复杂的操作语句通过视图封装起来，屏蔽了底层的复杂操作。
  3. 保证数据安全性， 在创建视图时加入一些条件限制，可以给用户授予表的特定部分的访问权限而不是整个表的访问权限，保证数据的安全。
  4. 连接分区数据。 为了提升性能，许多数据库在设计时将 大表分成许多小块，使用视图可以很方便的将分区表数据联合查询出来，像在一张表中那样简单。

存储过程

概念

有些功能通过一条 SQL语句不可能实现，需要多条语句一起执行， 这时候就需要一个叫做存储过程的功能。

存储过程简单来说，就是为以后的使用而保存的一条或多条MySQL语句的集合。

使用示例

语法解析：

  • 存储过程名字前面 要加关键字PROCEDURE,

  • 存储过程名字后要加括号()

  • 调用存储过程应该使用 :call procedure_name

优点

  1. 简单： 使复杂的逻辑变得简单，只需要调用一下即可。
  2. 数据完整性： 不需要建立一系列的处理步骤，保证了数据的完整性。
  3. 简化变动管理： 变动时可以只修改底层逻辑实现，不影响调用者。
  4. 提高性能： 使用存储过程比单独的SQL语句要快。

INSERT触发器

有一个名为new的表，用于记录新插入的数据。

DELETE触发器

使用DELETE触发器前，需要先建立一张对应的 保存 删除内容的表。

有一张OLD表，用于保存刚刚删除的记录。

OLD表中的 值是只读的，不允许更新。

UPDATE触发器

UPDATE触发器在UPDATE语句执行之前或之后执行。

  • 在UPDATE触发器代码中，可以引用一个名为OLD的虚拟表访问UPDATE语句前的值，引用一个名为NEW的虚拟表访问新

更新的值；

  • 在BEFORE UPDATE触发器中，NEW中的值可能也被更新（允许更改将要用于UPDATE语句中的值）；

语法分析

CREATE TRIGGER+名称: 创建 触发器。

AFTER INSERT： 在插入之后触发

BEFORE INSERT: 在删除之前触发

游标

什么是游标

游标是 存储在数据库上的数据库查询，它不是一个select语句，而是select语句查询出来的结果集。

存储了游标后，应用程序可以根据需要滚动或者浏览其中的数据。

主要用于交互式应用。
