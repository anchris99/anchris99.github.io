java IO

发表于 2020-09-03 分类于 java基础

概念

Java IO 是一套Java用来读写数据（输入和输出）的API。

Java IO中的一个核心概念是流（Stream），从外部（包括磁盘文件、键盘、网络套接字）读入到内存中的数据序列称为输入流，从内存写入到外部设备的数据序列称为输出流。

按照流传输的单位，Java中的流分为两种：

  • 字节流： 最小单元是字节，多用于 二进制数据。
  • 字符流：数据流中最小的数据单元是字符。

NIO

NIO是引入的 高速的、 面向块的IO，弥补普通IO的不足（IO速度慢，耗费大）。

  • I/O 与 NIO 最重要的区别是数据打包和传输的方式，I/O 以流的方式处理数据，而 NIO 以块的方式处理数据。I/O是阻塞的，NIO非阻塞。

  • 面向流的 I/O 一次处理一个字节数据：一个输入流产生一个字节数据，一个输出流消费一个字节数据。

    • 优点： 为流式数据创建过滤器非常容易，链接几个过滤器，以便每个过滤器只负责复杂处理机制的一部分。
    • 缺点：面向流的 I/O通常相当慢。

  • NIO 利用Channel 复制文件实例

关于：buffer.flip()的说明： 该操作是 进行读写切换（先读后写），整个缓冲区的变化：

① 新建一个大小为 8 个字节的缓冲区，此时 position 为 0，而 limit = capacity = 8。

② 从输入通道中读取 5 个字节数据写入缓冲区中，此时 position 为 5，limit 保持不变。

③ 在将缓冲区的数据写到输出通道之前，需要先调用 flip() 方法，这个方法将 limit 设置为当前 position，并将 position 设置为 0。

④ 从缓冲区中取 4 个字节到输出缓冲中，此时 position 设为 4。

⑤ 最后需要调用 clear() 方法来清空缓冲区，此时 position 和 limit 都被设置为最初位置。
