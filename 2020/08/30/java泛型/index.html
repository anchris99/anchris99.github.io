java泛型

发表于 2020-08-30 更新于 2020-08-31 分类于 java基础

泛型的概念

  • 泛型，即“参数化类型”，就是将类型由原来的具体的类型参数化，类似于方法中的变量参数，此时类型也定义成参数形式（可以称之为类型形参），然后在使用/调用时传入具体的类型（类型实参）。

  • 泛型的目的是在不创建新的类型的情况下，通过泛型指定的不同类型来控制形参具体限制的类型。也就是说在泛型使用过程中，操作的数据类型被指定为一个参数 。

  • 泛型对于集合类尤其有用，它提供了编译期的类型安全，确保你只能把正确类型的对象放入集合中。在集合类中使用泛型机制编写的程序代码要比那些杂乱地使用Object 变量，然后再进行强制类型转换的代码具有更好的安全性和可读性。

通配符

类型通配符一般是使用?代替具体的类型参数。例如 List<?> 在逻辑上是List<String>``,List<Integer> 等所有 List<具体类型实参>的父类。

?代表的是 具体 的一种类型实参，而不是类型参数，和可以把?看成所有类型的父类，是一种真实的类型。

使用场景：

  • 具体类型不确定的时候，这个通配符就是 ? ；

  • 当 不需要使用类型的具体功能时，只使用通用方法时。那么可以用 ?通配符来表未知类型。

Java 泛型转换的事实

  • java虚拟机中没有泛型，只有普通类、普通方法
  • 所有类型参数在编译后，都会被限定类型（原始类型） 替换。
  • 用 合成桥方法 来保持多态
  • 为保持安全性，有时需要插入强制类型转换。
