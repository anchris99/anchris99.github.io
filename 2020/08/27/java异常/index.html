java异常

发表于 2020-08-27 更新于 2020-08-28 分类于 java基础

什么是异常

​ 异常指出现的各种非正常的状况，如：文件找不到、网络连接失败、除0操作、非法参数等。异常是一个事件，它发生在程序运行期间 。

​ 当发生异常情况时，一个代表该异常的对象被创建并且在导致该异常的方法中被抛出，而该方法可以选择自己处理异常或者传递该异常。

详解

  1. Throwable分为两个分支：Error 和Exception
  • Error，指 java 运行时系统的内部错误和资源耗尽错误。应用程序不会抛出该类对象。 时候程序无法捕获或者 无法处理的错误， 出现了这样的错误，除了告知用户，剩下的就是尽力使程序安全的终止。
  • 另一个分支是Exception，是用户程序可能捕捉或者处理的异常情况 。
    • 按照发生时间分为：运行时异常(RuntimeException)和非运行时异常。
    • 按照是否受到检查分为：不受检查异常（Unchecked Exception）和检查异常（Checked Exception）。
  1. RuntimeException一般在 程序运行期间 发生，编译时期是无法获悉是否会发生异常的，因此若不处理也可以编译通过，因为它不受检查。 而非运行时异常，是可以提前预见到的，必须在代码中进行处理，否则 编译不会通过。

  1. 派生 于 Error 类 或 RuntimeException 类的所有异常称为非受查( unchecked ) 异常，所有其他的异常称为受查（ checked) 异常。

    • 出现 RuntimeException一定是程序编写出现了问题；

    • 而程序本身没有问题， 但由于像 I/O 错误这类问题导致的异常属于非运行时异常。

    所有的受查异常 一定要在程序中得到处理。要么使用try-catch语句进行捕获，要么用throws子句抛出，否则编译无法通过。

如何处理异常

抛出异常

对于异常情形， 无法继续运行下去，在当前环境下无法解决，所能做的就是**从当前环境中跳出**，并把问题提**交给上一级**环境，这就是抛出异常时所发生的事情。

抛出异常后：

  1. 像创建普通的java对象一样 将使用new在堆上创建一个异常对象；
  2. 从当前环境中弹出对异常对象的引用。此时，异常处理机制接管程序， 异常处理器继续执行程序， 它的任务是将程序从错误状态中恢复，以使程序要么换一种方式运行，要么继续运行下去。

捕获异常

​ 在方法抛出异常之后，运行时系统将转为寻找合适的异常处理器（exception handler）。当异常处理器所能处理的异常类型与方法抛出的异常类型相符时，即为合适的异常处理器。

  1. try catch语句
  1. try嵌套

嵌套try语句：try语句可以被嵌套。也就是说，一个try语句可以在另一个try块的内部。

  1. throw：应用于某个语句，抛出明确的异常。程序执行完throw语句之后立即停止；throw后面的任何语句不被执行，最邻近的try块用来检查它是否含有一个与异常类型匹配的catch语句

  2. throws子句: 应用于方法， 使该方法的 调用者 可以 保护它们自己 而不发生异常。

创建自己的异常

在程序中，可能会遇到任何标准异常类都没有能够充分地描述清楚的问题。 在这种情况下，创建自己的异常类就是一件顺理成章的事情了。 我们需要做的只是定义一个派生于Exception 的类，或者派生于 Exception 子类的类。

常见问题

比较throw和throws

  1. throws出现在方法函数头；而throw出现在函数体。

  2. throws表示出现异常的一种可能性，并不一定会发生这些异常；throw则是抛出了异常，执行throw则一定抛出了某种异常对象。执行到throw ，功能也就结束了

  3. 两者都是消极处理异常的方式（这里的消极并不是说这种方式不好），只是抛出或者可能抛出异常，但是不会由函数去处理异常，真正的处理异常由函数的上层调用处理。

try-catch-finally省略

try-catch-finally 其中catch 和 finally 都可以被省略，但是不能同时省略，也就是说有 try 的时候，必须后面跟一个 catch 或者 finally。

try-catch-finally 中return

  • try-catch-finally 中，如果 catch 中 return 了，finally 还会执行吗？

finally 一定会执行，即使是 catch 中 return 了，catch 中的 return 会等 finally 中的代码执行完之后，才会执行。
