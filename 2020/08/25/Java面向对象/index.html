所有类的超类-Object

概述

类 Object 是类层次结构的根类。每个类都使用 Object 作为超类。所有对象（包括数组）都实现这个类的方法。

常用的方法

hashCode()

  • hashCode() 返回哈希值，而 equals() 是用来判断两个对象是否等价。
  • 等价的两个对象散列值一定相同，但是散列值相同的两个对象不一定等价。
  • hashCode和equals方法一同重写。在覆盖 equals() 方法时应当总是覆盖 hashCode() 方法，目的：保证等价的两个对象哈希值也相等。

应用：HashSet 和HashMap等集合类使用了 hashCode() 方法来计算对象应该存储的位置，需要重写hashCode和equals方法。

  • 如果 类不重写 hashCode方法，那么其值 默认是 对象的地址值。
  • String类重写了hashCode, 而 StringBUffer没有重写 hashCode方法。

equals()

  1. equals方法五大特点：
    1. 自反性 x.equals(x);
    2. 对称性 (a.equals(b)则(b.equals(a)))
    3. 传递性 if (x.equals(y) && y.equals(z)) 则x.equals(z);
    4. 一致性 不论调用多少次，结果一致
    5. 空类型返回 false 对任何不是 null 的对象 x 调用 x.equals(null) 结果都为 false
  2. equals标准步骤
    1. 检查二者是否是同一对象
    2. 检查参数对象是否为空， 是，则返回 false
    3. 检查二者是否是同一类
    4. 将Object类进行转换
    5. 检查对应的属性是否相同

注意事项：equals方法 括号内的 参数必须是Object类型，否则就不是对 Object类方法的重载了。

  1. == 和 equals()
  • 对于基本类型，== 判断两个值是否相等 。
  • 对于引用类型，== 判断两个变量是否引用同一个对象，而 equals() 判断引用的对象是否等价。

clone()

  • clone() 是 Object 的 protected 方法，它不是 public，若不显式去重写 clone()，其它类就不能直接去调用该类实例的 clone() 方法。
  • clone 方法 的类必须继承Cloneable接口
  1. 重写 clone方法
  1. 深浅拷贝

  • 浅拷贝就是创建一个新对象，新对象的属性和原来对象完全相同，对于非基本类型属性，仍指向原有属性所指向的对象的内存地址。

    克隆操作默认就是浅拷贝。

  • 深拷贝就是 创建一个新对象，属性中引用的其他对象也会被克隆，重新拷贝一份，不再指向原有对象地址。

  1. clone 来复制一个对象不仅复杂，且涉及强制类型转换，通常推荐使用 构造方法 来复制一个对象。

面向对象的特性

面向对象的目的：解决软件系统的可扩展性、可维护性、可重用性；

（1）封装：

通常认为封装是把数据和操作数据的方法绑定起来，对数据的访问只能通过已定义的接口。简单说，封装就是隐藏一切可隐藏的东西，只向外界提供最简单的编程接口。

（2）继承：

继承是从已有类得到继承信息创建新类的过程。提供继承信息的类被称为父类（超类、基类）；得到继承信息的类被称为子类（派生类）。继承让变化中的软件系统有了一定的延续性。

（3）多态：

多态性是指允许不同子类型的对象对同一消息作出不同的响应。

方法重载（overload）实现的是编译时的多态性（也称为前绑定）

方法重写（override）实现的是运行时的多态性（也称为后绑定）。

运行时的多态是面向对象最精髓的东西，含义：定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编译时并不确定，而是在程序运行期间才确定。不用修改源代码，就可以让引用变量绑定到各种不同的对象上。

要实现多态需要做两件事：

  • 第一：方法重写（子类继承父类并重写父类中已有的或抽象的方法）；

  • 第二：对象造型（用父类型引用指向子类型对象，这样同样的引用调用同样的方法就会根据子类对象的不同而表现出不同的行为）。

抽象类和接口

  1. 概念、使用实例
  • 抽象类：类中有未实现的方法，类需要用abstract修饰，抽象方法也要用abstract修饰，不可直接实例化对象，被继承后需要实现抽象方法（若不实现，则仍然为抽象类）。 实现类用extends继承抽象类。
  • 接口：使用interface修饰。实现类 使用 implements继承。
    • 可以有 常量 、 抽象方法、 默认方法、 静态方法 ， 不可以有 实例域。
    • 方法默认public属性， 常量默认 为public static final

从 Java 8 开始，接口也可以拥有默认的方法实现。在 Java 8 之前，如果一个接口想要添加新的方法，那么要修改所有实现了该接口的类，让它们都实现新增的方法。

  1. abstract类和interface对比
    1. Java 8之前, implementations: Java interface are implicitly abstract and cannot have implementations. A Java abstract class can have instance methods that implements a default behavior. Java 8及之后，interface中可以有default和public static方法
    2. Variables declared in a Java interface is by default public static final. An abstract class may contain non-final variables.
    3. Methods of a Java interface are public by default. A Java abstract class can have the usual flavors of class members like private, protected, etc..
    4. Java interface should be implemented using keyword “implements”; A Java abstract class should be extended using keyword “extends”.
    5. An interface can extend another Java interface only, an abstract class can extend another Java class and implement multiple Java interfaces.
    6. A Java class can implement multiple interfaces but it can extend only one abstract class.
    7. In comparison with java abstract classes, java interfaces are slow as it requires extra indirection.
  • Java8后引入default和 static final的目的：

The reason we have default methods in interfaces is to allow the developers to add new methods to the interfaces without affecting the classes that implements these interfaces.

在更改interface的时候，可以不影响已经实现的实现类。

重写与重载

1. 重写（Override）

存在于继承体系中，指子类实现了一个与父类在方法声明上完全相同的一个方法。

重写有以下三个限制：

  • 子类方法的访问权限必须大于等于父类方法；
  • 子类方法的返回类型必须是父类方法返回类型或为其子类型。
  • 子类方法抛出的异常类型必须是父类抛出异常类型或为其子类型。

在调用一个方法时，先从本类中查找看是否有对应的方法，如果没有再到父类中查看，看是否从父类继承来。否则就要对参数进行转型，转成父类之后看是否有对应的方法。总的来说，方法调用的优先级为：

  • this.func(this)
  • super.func(this)
  • this.func(super)
  • super.func(super)

2. 重载（Overload）

同一个类中，一个方法与已经存在的方法名称上相同，但是 参数类型、个数、顺序至少有一个不同。

注意：返回值不可作为重载依据。
