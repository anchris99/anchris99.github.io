举例理解jvm指令和Java运行时数据区

发表于 2020-10-07 分类于 jvm

举例1

理解：

  1. 首先要知道 jvm stack 中的 栈帧中的 local variables表（局部变量表）和 Operand Stack（操作数栈） 的概念：

    • 每个线程对应一个 jvm stack。
    • 线程中的每个方法对应一个栈帧
    • 栈帧包括四部分： local variables（局部变量表） + operand stack)(操作数栈)+ dynamic linking(动态连接)+ return address（返回地址）。理解这种现象 主要先理解栈帧中的前两个部分。

  2. 对应的指令码的意思：

    • bipush指令对应 将数据压入operand stack，istore是从operand stack中取数据赋值到local variables， 而iload操作是 从 local variables取出数据 加载到操作数栈operand stack。
    • iinc 1 by 1是指 对 i加1操作： Increment local variable by constant 其在官方文档中描述如下：

    Operation

    Increment local variable by constant

    Forms

    iinc = 132 (0x84)

    Operand Stack

    No change

    Description

    The index is an unsigned byte that must be an index into the local variable array of the current frame The const is an immediate signed byte. The local variable at index must contain an int. The value const is first sign-extended to an int, and then the local variable at index is incremented by that amount.

    注意到 该操作是在 local variables表中进行的，并不涉及 操作数栈。

  3. 观察两个案例对应的 指令， 第一个是先 load再 进行iinc，然后又istore， 由于iinc是在local variables中进行的，所以operand stack中的 i值没有改变， 然后又赋值回了 local variables中， 变回了8

  4. 而第二个例子中， 第一个是先iinc再 进行load，然后 store，由于是先进行了iinc操作，所以加载到栈中的 数据已经是加一后 的值（9）， 再赋值回来，仍然是9。

  5. 还有一点： 在Java中看起来从一个局部变量到另外一个局部变量是一步操作完成的，其实在jvm指令码层面是两步， 比如：

举例2

new 之后发生了什么

对应的字节码指令如下：

new :创建一个新的对象 Create new object， 此时在 operand stack中存放的是 新对象的引用。 新对象此时是默认值（零值）

dup： Duplicate the top value on the operand stack and push the duplicated value onto the operand stack.

invokespecial：调用 特殊方法（这里是初始化方法）（会消耗掉栈顶 duplicate后的元素）

astore_1: 将 operand stack 栈顶元素存入 local variables局部变量表。

由上面的内容也进一步理解了 DCL模式时为什么要加 volatile， 若不加volatile则 可能会发生指令重排序， 那么 astore指令就可能在 invokespecial之前那执行，那么另外一个线程就有可能使用到对象 半初始化的数据（也就是说执行完了 new（此时值是默认值） 尚未执行初始化操作）。因此要用volatile修饰变量。
