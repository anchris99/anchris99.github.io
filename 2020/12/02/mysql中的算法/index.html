mysql中的算法

发表于 2020-12-02 分类于 数据库

关联查询算法

MySQL 中表之间执行连接主要有两种算法：

  • Nested-Loop Join 算法
  • Block Nested-Loop Join 算法

Nested-Loop Join

译文：嵌套循环连接算法

主要思想：

一个简单的 Nested-Loop Join算法一次一行循环地从 驱动表 中读取行，在这行数据中取到关联字段，根据关联字段在 被驱动表 里取出满足条件的行，然后取出两张表的结果合集。

如果在被驱动表中这个关联字段没有索引，那么每次取出驱动表的关联字段在被驱动表查找对应的数据时，都会对被驱动表做一次全表扫描，成本是非常高的（比如驱动表数据量是 m，被驱动表数据量是 n，则扫描行数为 m * n ）。

Block Nested-Loop Join

译文：阻塞嵌套循环连接

算法思想：

Block Nested-Loop Join：把 驱动表 的数据读入到 join_buffer 中，然后扫描被驱动表，把被驱动表每一行取出来跟 join_buffer 中的数据做对比，如果满足 join 条件，则返回结果给客户端。

驱动被驱动表

  • 驱动表：两个表中先进行查询扫描的表，在扫描出的数据基础上对另外一个表的数据进行筛选

  • 被驱动表：后扫描数据的表，根据驱动表提供的数据条件进行 查询。

JOIN类型 驱动表
left join join左边的表
right join join右边的表
inner join 数据量较小的表

使用场景

使用NLJ：

被驱动表的关联字段上有索引，使用NLJ

使用BNLJ：

被驱动表关联字段上没有索引，使用BLJ

原因分析

为什么根据被驱动表上关联字段有无索引来判断使用哪个算法？举个例子来说明：

有两个表t1,t2， t1 有10条数据， t2有1000条数据， 选择较小的表t1作为驱动表。

  • t2(被驱动表)关联字段有索引的时候，我们使用 NLJ算法，一次一行读取t1的数据，根据读出的数据在t2里面进行筛选对比，由于t2的关联字段有索引，所以查询是非常快的，这个时候我们总计需要磁盘扫描 20次：2*t1数据量大小。

  • t2(被驱动表)关联字段没有索引时，若采用NLJ算法，由于没有索引，所以对于t1的每行数据，都要在t2里逐行对比,此时总计要查询 10*1000 = 10000次磁盘扫描，这种耗费是非常大的。所以采用BNLJ算法， 首先将驱动表t1的数据 10条扫描到内存中，然后逐行扫描被驱动表的数据，每行数据都和已经读取到内存中的t1的10条数据进行比对，因此比对 10*1000==10000次。 注意，这是在内存中的比对，而使用NLJ是在磁盘扫描这么多次。

    使用BNLJ在这种情况下 磁盘扫描的 次数= 10+1000=1010次，可见性能比 NLJ还是高很多的。

join查询性能优化的启示

由上面的分析，为了提高join的查询性能，我们可以有以下几个措施：

  1. 给被驱动表关联字段建立索引

这样可以避免使用BNLJ算法，减少大量磁盘扫描耗费时间。

  1. 使用数据量较小的表作为驱动表

在NLJ算法中，磁盘扫描的次数为：2*驱动表数据量,因此，驱动表的数据量越小越好。

排序算法

使用order by语句的时候就是对查询的数据进行排序。MySQL主要的排序原理有两个： 单路排序和双路排序。

单路排序

是一次性取出满足条件行的所有字段，然后在sort buffer中进行排序；

举例：

单路排序的详细过程：

  1. 从索引 a 找到第一个满足 a = 1000 条件的主键 id
  2. 根据主键 id 取出整行，取出 a、c、d 三个字段的值，存入 sort_buffer 中
  3. 从索引 a 找到下一个满足 a = 1000 条件的主键 id
  4. 重复步骤 2、3 直到不满足 a = 1000
  5. 对 sort_buffer 中的数据按照字段 d 进行排序
  6. 返回结果给客户端

双路排序

首先根据相应的条件取出相应的排序字段和可以直接定位行数据的行 ID，然后在 sort buffer 中进行排序，排序完后需要再次回表查询，按顺序将对应的数据取出。

对应上面的例子，双路排序的过程：

  1. 从索引 a 找到第一个满足 a = 1000 的主键 id
  2. 根据主键 id 取出整行，把排序字段 d 和主键 id 这两个字段放到 sort buffer 中
  3. 从索引 a 取下一个满足 a = 1000 记录的主键 id
  4. 重复 3、4 直到不满足 a = 1000
  5. 对 sort_buffer 中的字段 d 和主键 id 按照字段 d 进行排序
  6. 遍历排序好的 id 和字段 d，按照 id 的值回到原表中取出 a、c、d 三个字段的值返回给客户端

对比

二者最主要的区别在于：单路排序选的时候会在sort_buffer中存储所有要查询的字段，然后根据排序字段进行排序，这是一次性装入内存的。 而双路排序，第一装入内存的只是：标记某行的 主键ID+排序的字段，将排序字段排序后，再根据排序后的顺序回到表中进行相应数据的查询，需要执行两次才能得到符合条件的数据。

  • 单路优点：不需要回表，执行效率高。

  • 单路缺点：加入内存的数据项目过多，导致可加入的数据条目较少，一次可排序的项目少。

  • 双路优点：每条数据项目占用空间小，一次加载的数据项目数量多，一次可以排序的数据条目较大。

  • 双路缺点：需要回表，浪费时间。

max_length_for_sort_data

可以调整参数，控制出现 双路排序和 单路排序的阈值，这个参数就是max_length_for_sort_data，当排序字段大于它时，使用 双路排序，反之使用单路排序。

排序的位置

MySQL 中的 Filesort 并不一定是在磁盘文件中进行排序的，也有可能在内存中排序，内存排序还是磁盘排序取决于 排序的数据大小和 sort_buffer_size 配置的大小。

  • 如果 “排序的数据大小” < sort_buffer_size: 内存排序
  • 如果 “排序的数据大小” > sort_buffer_size: 磁盘排序
